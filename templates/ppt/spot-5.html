Notion

프로젝트 전체 문서화, 회의록, API 명세서, WBS 일정표를 작성·관리합니다. 실시간 동시 편집으로 팀원 간 정보 격차를 완전히 해소합니다.

GitHub

소스코드 원격 저장소 및 Pull Request 기반의 코드 리뷰를 진행합니다. 이슈 트래킹과 CI 연동으로 변경 이력을 투명하게 관리합니다.

SourceTree

시각적 GUI로 브랜치 흐름을 한눈에 파악하고, 커밋·병합·충돌 해결 등 로컬 Git 작업을 CLI 없이 직관적으로 수행합니다.

Jira (Kanban)

Epic(에픽)-Story(스토리)-Task(태스크) 기반으로 WBS 일정을 수립하고, 병목(Bottleneck) 구간 트래킹을 통해 스프린트 효율을 극대화합니다.

이미지 처리 및 비동기 AI 파이프라인

사용자가 업로드한 원본 데이터의 메타데이터를 파쇄하고 규격을 정제하여, Threading을 통해 메인 프로세스 중단 없이 지능형 AI 판독을 비동기로 수행합니다.

STAGE 1

Pillow — 데이터 정제

Metadata Shredding 개인정보 보호를 위해 촬영 기기 정보 및 EXIF 데이터를 즉시 파쇄
Format Standard HEIC/HEIF 등 모바일 전용 포맷을 AI 호환 JPG로 자동 변환 및 85% 압축
STAGE 2

OpenCV/YOLO — AI 시각화

Bounding Box Annotation YOLO 추론 결과를 OpenCV를 통해 이미지/영상 위에 시각적 객체로 매핑
Dynamic Frame Compute 영상의 프레임 단위 분해 및 박스가 합성된 고효율(H.264) 영상 재인코딩
STAGE 3

Data — 정형화

NumPy 배열화 이미지를 숫자 행렬(Matrix)로 변환, AI 모델 입력 데이터로 활용
GPS 연동 & DB 저장 위치 좌표·이미지 매칭 후 TiDB에 저장, 구역별 쿼리 지원
정규화 픽셀 값(0~255)을 0~1 범위로 스케일링, YOLO 학습 최적 상태
STAGE 4

Threading — 비동기

Non-Blocking AI 사용자 제출 즉시 응답, 별도 스레드에서 YOLO 판독 수행
DB 자동 반영 AI 결과를 DB에 비동기 커밋, 사용자 경험 저해 없이 처리
